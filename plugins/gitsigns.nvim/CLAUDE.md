# gitsigns.nvim

## Project Overview

gitsigns.nvim is a super-fast Git status tracking plugin implemented entirely in Lua. It shows marks for added, removed, and modified lines in Neovim's sign column, and provides rich hunk operations and preview capabilities.

**Note:** There is no `plugin/` directory — the `:Gitsigns` command is registered programmatically via `nvim_create_user_command` in `setup_cli()` (`gitsigns.lua`).

Features:
- **Signs** — Display marks for added, modified, and deleted lines in the sign column (with optional counts, line-number highlighting, line highlighting, and word-diff)
- **Hunk operations** — Stage, undo-stage, reset, and preview hunks
- **Navigation** — Jump quickly between hunks (`]c`/`[c`)
- **Diff viewing** — Diff the current file against the index or any commit
- **Live blame** — Show git blame for the current line via virtual text
- **Hunk text object** — Select a hunk with `ih`
- **Virtual lines** — Display deleted/modified lines inline via virtual lines
- **File tracking** — Automatically follow files moved in the index (`git mv`)
- **CLI interface** — Full Tab-completion via `:Gitsigns`
- **Statusline integration** — Provides `b:gitsigns_status`, `b:gitsigns_status_dict`, and `b:gitsigns_head`
- **Staged signs** — Optionally show signs for already-staged hunks (`_signs_staged_enable`)

## Core Modules

### `gitsigns` (top-level entry, `lua/gitsigns.lua`)

- `M.setup(cfg)` — Configure and start Gitsigns (async).
- **Metatable dispatch**: via `__index`, `M.foo()` looks up `gitsigns.attach.foo`, then `gitsigns.actions.foo`, and (only when `config.debug_mode == true`) `gitsigns.debug.foo`.
- `setup_cli()` registers the `:Gitsigns` user command and its Tab-completion.
- `setup_attach()` registers `BufRead`/`BufNewFile`/`BufWritePost` autocommands to auto-attach to buffers.
- `setup_cwd_head()` tracks the current branch and watches `.git/HEAD` for branch switches.
- Augroup name: `gitsigns` (created in `setup` via `nvim_create_augroup`).

```lua
require('gitsigns').setup {
  signs = { add = { text = '┃' }, ... },
  signcolumn = true,
  ...
}
```

### `gitsigns.config` (`lua/gitsigns/config.lua`)

Config is backed by a lazy metatable — fields are read from `M.schema` and resolved on first access.

- `M.build(user_config)` — Resolves defaults, handles deprecated fields, validates, and populates `M.config` (called from `gitsigns.setup()`).
- `M.schema` — Schema definition table; each entry has `type`, `default`, `description`, and optional `deprecated`, `deep_extend`, `refresh`, `default_help` fields.
- Access config fields directly as `config.foo` (no `config.get()` / `config.setup()` functions exist).
- Undocumented (internal) schema fields are prefixed with `_` (e.g. `_extmark_signs`, `_inline2`, `_threaded_diff`, `_git_version`, `_verbose`, `_test_mode`, `_signs_staged_enable`, `_refresh_staged_on_update`).
- `current_line_blame_formatter_opts` is **deprecated**; `yadm` is **deprecated** (use `_on_attach_pre`).

### `gitsigns.attach` (`lua/gitsigns/attach.lua`)

- `M.attach(bufnr, ctx, trigger)` — Attach Gitsigns to a buffer; calls `config.on_attach(bufnr)` so users can set keymaps.
- `M.detach(bufnr, keep_signs)` — Detach from a buffer.
- `M.detach_all()` — Detach from all buffers (called on `VimLeavePre`).
- Handles fugitive:// and gitsigns:// buffers transparently via URI parsing (`parse_fugitive_uri`, `parse_gitsigns_uri`).
- Tries `config.worktrees` entries when the normal attach fails (detached working trees).

### `gitsigns.actions` (`lua/gitsigns/actions.lua`)

User-callable API:

- **Hunk staging/reset**: `stage_hunk(range?, opts?)`, `reset_hunk(range?, opts?)`, `undo_stage_hunk()`, `stage_buffer()`, `reset_buffer()`, `reset_buffer_index()`
- **Navigation**: `nav_hunk(direction, opts?)` (preferred), `next_hunk(opts?)`, `prev_hunk(opts?)` (backwards-compat wrappers)
- **Preview**: `preview_hunk()`, `preview_hunk_inline()`
- **Blame**: `blame_line(opts?)`
- **Diff**: `diffthis(base?, opts?)`, `show(revision)`
- **Selection**: `select_hunk()`, `get_hunks(bufnr)`
- **Base change**: `change_base(base, global)`, `reset_base(global)`
- **Lists**: `setqflist(target?, opts?)`, `setloclist(nr?, target?)`
- **Toggle**: `toggle_signs()`, `toggle_numhl()`, `toggle_linehl()`, `toggle_word_diff()`, `toggle_deleted()`, `toggle_current_line_blame()`
- **Misc**: `refresh()`, `get_actions()`
- `stage_hunk` and `reset_hunk` are wrapped with `mk_repeatable` so they work with `.`.

### `gitsigns.async` (`lua/gitsigns/async.lua`)

Coroutine-based async framework built on libuv.

- `M.wrap(argc, func)` — Convert a callback-style function into an async function.
- `M.create([argc,] func)` — Create an async function (when called from a non-async context, cannot return a value).
- `M.wait(argc, func, ...)` — Inside an async function, wait for a callback-style function to complete.
- `M.run(func, ...)` — Run a function in a new coroutine (fire-and-forget).
- `M.scheduler` — Yield to the Neovim scheduler so the API can be called (`M.wrap(1, vim.schedule)`).
- `M.running()` — Returns whether the current execution context is async (i.e. inside a coroutine managed by this module).
- Handle objects returned by `run`/`create` provide `cancel(cb)` and `is_cancelled()`.

### `gitsigns.git` (`lua/gitsigns/git.lua`)

Git command wrappers.

- `M.Obj` (`Gitsigns.GitObj`) — File-level Git info (relpath, object_name, mode_bits, has_conflicts, repo).
- `M.Repo` (`Gitsigns.Repo`) — Repo-level info (gitdir, toplevel, detached, abbrev_head, username).
- `M.Repo` derives from `Gitsigns.RepoInfo`.
- `M.get_repo_info(cwd)` — Get repo info for a working directory.
- `asystem = async.wrap(3, system)` — Async wrapper around `vim.system` (polyfilled for older versions via `system.compat`).

### `gitsigns.git.version` (`lua/gitsigns/git/version.lua`)

- `M.check(version_tuple)` — Returns true if the detected git version is >= the given version `{major, minor?, patch?}`.
- `M.set_version()` — Auto-detects the git version (runs `git --version` once).

### `gitsigns.manager` (`lua/gitsigns/manager.lua`)

Core buffer update/render scheduler.

- `M.update(bufnr)` — Compute diff, update signs (throttled per buffer via `throttle_by_id`).
- `M.update_debounced(bufnr)` — Debounced variant.
- `M.schedule(bufnr, check_compare_text?)` — Prepare a buffer for update (returns false if buffer is detached/invalid).
- `M.on_lines(bufnr, first, last_orig, last_new)` — Incremental sign update callback.
- `M.detach(bufnr, keep_signs)`, `M.reset_signs()`, `M.setup()`.

### `gitsigns.cache` (`lua/gitsigns/cache.lua`)

- `M.cache` — Per-bufnr cache table.
- `M.CacheEntry` — Exact class with: `bufnr`, `file`, `base?`, `compare_text?`, `hunks?`, `hunks_staged?`, `staged_diffs`, `gitdir_watcher?`, `git_obj`, `commit?`, `blame?`, `force_next_update?`, `compare_text_head?`.
- `CacheEntry:get_compare_rev(base)` — Returns the revision to compare against (`:0` for index, `commit^` for fugitive buffers, etc.).
- `CacheEntry:get_rev_bufname(rev)` — Returns a `gitsigns://` URI for the given revision.
- `CacheEntry:invalidate(all?)` — Drops cached hunks/blame (and optionally compare_text).
- `CacheEntry:get_blame(lnum, opts)` — Fetch blame for a line (cached).
- `CacheEntry:run_blame(lnum, opts)` — Run blame (with changedtick stabilization).

### `gitsigns.signs` (sign rendering)

- `lua/gitsigns/signs.lua` — Dispatcher; selects backend based on `config._extmark_signs` (default: `true` → `extmarks.lua`; `false` → `vimfn.lua`).
- `B.new(cfg, name)` — Construct a backend instance.
- Backend interface (`Gitsigns.Signs`): `_new(cfg, hls, name)`, `add(bufnr, signs)`, `remove(bufnr, start_lnum?, end_lnum?)`, `contains(bufnr, start, last)`, `on_lines(bufnr, first, last_orig, last_new)`, `reset()`.
- `Gitsigns.Sign` — `{ type: SignType, count?: integer, lnum: integer }`.

### `gitsigns.cli` (`lua/gitsigns/cli.lua` and `cli/argparse.lua`)

- `:Gitsigns <action> [args...]` — Runs an action by looking up in `actions`, `attach`, or (debug-only) `debug` modules.
- Tab-completion is driven by `M.complete(arglead, line)` and subcommand-specific completion functions.
- `argparse.lua` parses `--flag`, `key=value`, quoted values, and positional arguments.

### `gitsigns.status` (`lua/gitsigns/status.lua`)

- `M:update(bufnr, status)` — Sets `b:gitsigns_head`, `b:gitsigns_status_dict`, `b:gitsigns_status` (formatted by `config.status_formatter`), and fires `User GitSignsUpdate`.
- `M:clear(bufnr)` — Clears all status variables.
- Exposes the `Gitsigns.StatusObj` class: `{ added, removed, changed, head, root, gitdir }`.

### `gitsigns.highlight` (`lua/gitsigns/highlight.lua`)

Defines all highlight groups (`GitSignsAdd`, `GitSignsChange`, `GitSignsDelete`, `GitSignsTopdelete`, `GitSignsChangedelete`, `GitSignsUntracked`, `GitSignsStaged*`, `GitSignsCurrentLineBlame`, plus `Nr`/`Ln`/`Inline`/`VirtLn` variants).

### Other Modules

- **`gitsigns.diff`** — Dispatch between `diff_int` (in-process via `vim.diff`) and `diff_ext` (external git-diff) based on `config.diff_opts.internal`.
- **`gitsigns.diffthis`** — Implements `:Gitsigns diffthis [base]` and `:Gitsigns show <revision>` (creates a scratch buffer showing the file at a given revision).
- **`gitsigns.current_line_blame`** — Virtual-text blame at end of current line (enabled by `config.current_line_blame`).
- **`gitsigns.watcher`** — Watches `.git/` dir via libuv `fs_event`, with debounced 200ms update handler. Tracks moved files in the index.
- **`gitsigns.popup`** — Floating-window helpers (`create_win`, auto-sizing, border handling).
- **`gitsigns.repeat`** — `mk_repeatable(fn)` — wraps a function so it becomes repeatable via `.` (sets `@:` via `operatorfunc` and `repeat#set`).
- **`gitsigns.message`** — `warn`, `error`, `error_once` wrappers around `vim.notify`.
- **`gitsigns.debug`** — `dump_cache()`, `debug_messages(noecho)`, `clear_debug()` (only available when `config.debug_mode == true`).
- **`gitsigns.debug.log`** — Leveled logging:

  | Function    | Condition                     |
  |-------------|-------------------------------|
  | `dprint`    | `debug_mode == true`          |
  | `dprintf`   | `debug_mode == true`          |
  | `vprint`    | `debug_mode && verbose`       |
  | `vprintf`   | `debug_mode && verbose`       |
  | `eprint`    | always (raises error)         |
  | `eprintf`   | always (raises error)         |
  | `assert`    | always (raises error if false)|

- **`gitsigns.system`** — Wraps `vim.system` or falls back to `system/compat.lua` (libuv-based polyfill for older Neovim).
- **`gitsigns.util`** — Utility functions including `tmpname()`, `expand_format()`, `buf_lines()`, `set_lines()`, path helpers, BOM handling.

### Install

require('gitsigns').setup()

### Default Configuration

```lua
require('gitsigns').setup {
  signs = {
    add          = { text = '┃' },
    change       = { text = '┃' },
    delete       = { text = '▁' },      -- Note: default is '▁' (not '_')
    topdelete    = { text = '▔' },      -- Note: default is '▔' (not '‾')
    changedelete = { text = '~' },
    untracked    = { text = '┆' },
  },
  signcolumn      = true,
  numhl           = false,
  linehl          = false,
  word_diff       = false,
  show_deleted    = false,
  sign_priority   = 6,
  update_debounce = 100,
  status_formatter = nil,              -- uses built-in formatter
  max_file_length  = 40000,
  preview_config   = { border = 'single', style = 'minimal', relative = 'cursor', row = 0, col = 1 },
  auto_attach           = true,
  attach_to_untracked   = false,
  current_line_blame    = false,
  current_line_blame_opts = {
    virt_text          = true,
    virt_text_pos      = 'eol',       -- 'eol' | 'overlay' | 'right_align'
    delay              = 1000,
    ignore_whitespace  = false,
    virt_text_priority = 100,
  },
  -- Note: default is ' <author>, <author_time> - <summary> ' (note surrounding spaces)
  current_line_blame_formatter = ' <author>, <author_time> - <summary> ',
  current_line_blame_formatter_opts = { relative_time = false },  -- DEPRECATED
  current_line_blame_formatter_nc  = ' <author>',                -- for uncommitted lines

  -- Diff options (derived from 'vim.opt.diffopt' by default)
  diff_opts = {
    algorithm           = 'myers',
    internal            = false,
    indent_heuristic    = false,
    vertical            = true,
  },

  -- Base revision to diff against (nil = index)
  base = nil,

  -- Count characters for sign counts (when show_count = true)
  count_chars = { [1]='1', [2]='2', ..., ['+']='>' },

  -- Detached working trees (array of {toplevel, gitdir})
  worktrees = nil,

  -- Asynchronous attach hook (used for detached worktrees)
  _on_attach_pre = nil,

  -- Called when attaching to a buffer; use to set keymaps
  on_attach = function(bufnr) end,

  -- Integrate with trouble.nvim for setqflist/setloclist
  trouble = true,   -- auto-detects if trouble is installed

  -- Internal / undocumented
  _extmark_signs         = true,   -- use extmarks for signs
  _inline2               = true,   -- enhanced preview_hunk_inline
  _threaded_diff         = true,   -- run diffs on a separate thread
  _signs_staged_enable   = false,  -- show signs for staged hunks
  _refresh_staged_on_update = false,
  debug_mode             = false,
}
```

### Keymaps Example

See README.md for the canonical setup. Use `on_attach` to register buffer-local keymaps:

```lua
on_attach = function(bufnr)
  local gitsigns = require('gitsigns')
  local function map(mode, l, r, opts)
    opts = opts or {}
    opts.buffer = bufnr
    vim.keymap.set(mode, l, r, opts)
  end

  map('n', ']c', function() gitsigns.nav_hunk('next') end)
  map('n', '[c', function() gitsigns.nav_hunk('prev') end)
  map('n', '<leader>hs', gitsigns.stage_hunk)
  map('n', '<leader>hr', gitsigns.reset_hunk)
  map('n', '<leader>hp', gitsigns.preview_hunk)
end
```

## Dependencies

### Required

- **Neovim >= 0.8.0** — recommended >= 0.9.0.
- **git** — must be available in the system PATH.

### Optional

- **[vim-fugitive](https://github.com/tpope/vim-fugitive)** — enables hunk display in fugitive revision buffers.
- **[trouble.nvim](https://github.com/folke/trouble.nvim)** — optional; when installed, `setqflist`/`setloclist` opens Trouble.

### Integrations

- **[vim-fugitive]**: When viewing revision buffers (`:0Gclog`), attaches with the base set to the parent commit so the signs reflect the changes introduced by that revision.
- **[trouble.nvim]**: If `config.trouble` is true (default: true if installed), opens Trouble for `setqflist`/`setloclist`.
- **[lspsaga.nvim]**: Can show gitsigns actions in its code-action UI when `code_action.extend_gitsigns = true`.

## Coding Conventions

### Code Style

Configured via `.stylua.toml`:

```toml
column_width = 100
line_endings = "Unix"
indent_type = "Spaces"
indent_width = 2
quote_style = "AutoPreferSingle"
call_parentheses = "Always"
```

### Naming Conventions

- Modules use hyphens for file names but are required with dots: `require('gitsigns.module-name')`.
- Classes/types use UpperCamelCase in their LuaCATS declarations (e.g. `Gitsigns.GitObj`, `Gitsigns.Repo`, `Gitsigns.Config`, `Gitsigns.Hunk.Hunk`).
- Private classes are declared local; public classes are exported via `M.Obj = Obj`.
- Internal config schema fields are prefixed with `_`.

### Type Annotations

Uses LuaCATS annotations throughout:

- `--- @class`, `--- @class (exact)` — class definitions
- `--- @field` — field definitions
- `--- @alias` — type aliases
- `--- @param` / `--- @return` — function signatures
- `--- @type` — variable type declarations
- `--- @cast` — type casts
- `--- @async` — async function markers
- `--- @diagnostic disable-next-line:...` — suppress specific diagnostics
- `--- @package` — marks a method as package-private
- `--- @generic F: function` — generic type parameters
- `---|` — sum/union type members

### Async Patterns

- All async work uses the in-house `async` module (coroutine + libuv).
- Convert callback-style → async via `async.wrap(argc, func)`.
- Create async entry points via `async.create([argc,] func)`.
- Wait for callbacks inside async via `async.wait(argc, func, ...)`.
- Yield to Neovim scheduler via `async.scheduler`.
- Call `uv.fs_*` etc. through `async.wrap` or use `vim.loop` directly.

### Logging

- Use `require('gitsigns.debug.log')`.
- `log.dprint(msg)` / `log.dprintf(fmt, ...)` — standard debug logs (only when `debug_mode == true`).
- `log.vprint(msg)` / `log.vprintf(fmt, ...)` — verbose logs (when `debug_mode && verbose`).
- `log.eprint(msg)` / `log.eprintf(fmt, ...)` — error logs (always active).
- The `__FUNC__` local at the top of async callbacks enables context-aware log output.

### Notifications

- Use `require('gitsigns.message').error_once(msg)` for one-shot errors.
- Use `warn(fmt, ...)` / `error(fmt, ...)` for immediate notifications (via `vim.notify`).

### Compatibility

- `local uv = vim.uv or vim.loop` — support both Neovim's old (`vim.loop`) and new (`vim.uv`) libuv APIs.
- `local system = vim.system or require('gitsigns.system.compat')` — polyfill for `vim.system`.
- `vim.fn.has('nvim-0.9')` is used to gate git version detection (auto-detected on < 0.9, assumed present on >= 0.9).
- Fugitive detection via `vim.fn.exists('*FugitiveReal')`.

### File Length Limit

- Files longer than `config.max_file_length` (default 40,000 lines) are skipped (see `attach.lua:197`).

### Statusline Variables

Attached buffers expose:
- `b:gitsigns_head` — current branch name.
- `b:gitsigns_status` — formatted status (e.g. `+3 ~2 -1`).
- `b:gitsigns_status_dict` — `{ added, removed, changed, head, root, gitdir }`.
- Autocmd `User GitSignsUpdate` fires when status changes (useful for reactive statuslines).

### Debug Mode

- `config.debug_mode = true` enables logging and exposes `dump_cache`, `debug_messages`, `clear_debug` via the metatable dispatch.
- `config._verbose = true` enables verbose logging (requires `debug_mode`).
