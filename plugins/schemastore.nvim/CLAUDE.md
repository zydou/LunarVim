# SchemaStore.nvim

## Project Overview

SchemaStore.nvim is a Neovim plugin that provides a Lua wrapper around the [SchemaStore](https://schemastore.org) catalog for use with LSP servers such as `jsonls` (vscode-langservers-extracted) and `yamlls` (yaml-language-server). It automatically converts the SchemaStore JSON catalog into a Lua module so that LSP configurations can reference predefined JSON/YAML schemas directly.

## Directory Structure

```
schemastore.nvim/
├── lua/schemastore/
│   ├── init.lua          # Main entry point; provides M.json.schemas() / M.yaml.schemas() API
│   ├── catalog.lua       # Auto-generated SchemaStore catalog (JSON -> Lua); do NOT edit manually
│   └── generate.lua      # Catalog generator; fetches from SchemaStore API and produces catalog.lua
├── scripts/
│   ├── generate          # Shell entry point for running the generator
│   ├── mulberry.lua      # Custom test framework (not plenary)
│   └── test              # Shell entry point for running tests
├── test/
│   ├── catalog.lua       # Tests for the catalog module
│   ├── generate.lua      # Tests for the generator module
│   └── schemastore.lua   # Tests for the main init module
├── Makefile              # `generate` and `test` targets
├── doc/schemastore.txt   # Vim help documentation
├── stylua.toml           # Stylua formatting configuration
└── README.md
```

## Core Modules

### `schemastore.init` — Main API

- `M.load()` — Returns the full catalog table: `{ json = { schemas = {...}, index = {...}, version = 1, ["$schema"] = "..." } }`
- `M.json.load()` — Returns the JSON portion of the catalog: `{ schemas = {...}, index = {...}, version = 1, ["$schema"] = "..." }`
- `M.json.get(name)` — Returns `(schema, nil)` for the schema with the given name, or `(nil, nil)` if not found
- `M.json.schemas(opts?)` — Returns the list of JSON schemas, with optional filtering/replacement/extension
- `M.yaml.schemas(opts?)` — Returns a YAML schema mapping in the form `{ url = fileMatch, ... }`

### `schemastore.catalog` — Catalog Data

Auto-generated file containing:
- `M.json.schemas` — Array of schema entries
- `M.json.index` — Lookup table mapping schema names to their array index
- `M.json.version` — Catalog version number
- `M.json["$schema"]` — SchemaStore catalog meta-schema URL

**Do not edit this file manually.** Regenerate it via `make generate`.

### `schemastore.generate` — Catalog Generator

- `M.setup(config)` — Configures the generator (base_url, output path, copyright notice)
- `M.run()` — Fetches the JSON catalog from the SchemaStore API and writes the Lua module to `M.config.out`

Default configuration:
- `base_url`: `https://www.schemastore.org/api`
- `out`: `/dev/stdout`
- `copyright_notice`: Apache 2.0 license header with attribution to Maddison Hellstrom and Mads Kristensen

## Usage

This plugin does not require a `setup()` call. Use it directly:

```lua
require('lspconfig').jsonls.setup {
  settings = {
    json = {
      schemas = require('schemastore').json.schemas(),
      validate = { enable = true },
    },
  },
}
```

For yamlls, you must disable the built-in schemaStore support:

```lua
require('lspconfig').yamlls.setup {
  settings = {
    yaml = {
      schemaStore = {
        enable = false,
        url = "",
      },
      schemas = require('schemastore').yaml.schemas(),
    },
  },
}
```

### `M.json.schemas()` Options (`SchemaOpts`)

| Option   | Type                          | Description |
|----------|-------------------------------|-------------|
| `select` | `string[]`                    | Return only schemas with the given names. Mutually exclusive with `ignore`. |
| `ignore` | `string[]`                    | Exclude schemas with the given names. Mutually exclusive with `select`. |
| `replace`| `table<string, SchemaEntry>`  | Replace existing schemas. Key is the schema name; value is the replacement `SchemaEntry`. The replacement's `name` field must match the original schema's name. |
| `extra`  | `SchemaEntry[]`               | Append custom schemas. If a schema with the same name already exists, the extra schema takes precedence. |

Notes:
- `select` + `replace` can be used together.
- `select` + `ignore` cannot be used together (throws an error).
- `extra` can be combined with `select`, `ignore`, or `replace`.

### `SchemaEntry` Structure

```lua
---@class SchemaEntry
---@field name string
---@field description string
---@field fileMatch string | string[]
---@field url string
```

To use a local schema file, use the `file://` scheme or a plain file path in the `url` field:

```lua
require('schemastore').json.schemas {
  extra = {
    {
      description = 'Local JSON schema',
      fileMatch = 'local.json',
      name = 'local.json',
      url = 'file:///path/to/your/schema.json',
    },
  },
}
```

## Dependencies

- **Runtime dependencies**: None
- **Build dependencies**: `curl`, `nvim` (both used by the generate script)
- **Consumed by**: `nvim-lspconfig` (jsonls/yamlls setup), LazyVim, LunarVim, and other Neovim configurations

## Build / Test

- `make generate` — Regenerate `lua/schemastore/catalog.lua` from the SchemaStore API
- `make test` — Run the test suite using the custom mulberry test framework

## Code Style

- Formatted with [stylua](https://github.com/JohnnyMorganz/StyLua) (configuration in `stylua.toml`)
- `catalog.lua` is wrapped with `-- stylua: ignore start` / `-- stylua: ignore end` to prevent formatting
- EmmyLua-style annotations are used (`---@class`, `---@field`, `---@param`)
- `catalog.lua` is auto-generated; never edit it manually
